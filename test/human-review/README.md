# Markdown Agent Comments V1 Human Review

Use this folder to review behavior before publishing.

## Files

- `agent-input.md` is the clean markdown file an agent may read. It should not contain labels that explain which cases should process or be ignored.
- `.generated/processed-output.md` is regenerated by `bun run test:review` and is ignored by git.
- `../human-review.test.js` asserts the scanner detects exactly the expected cases from `agent-input.md`.

## How To Review

Run:

```bash
bun run test:review
```

Then read:

- `test/human-review/agent-input.md`
- `test/human-review/.generated/processed-output.md`

## Expected Scanner Cases

These sections in `agent-input.md` should be detected:

- Opening copy: inline concrete edit for `@claude`
- Launch task: task checkbox update for `@agent`
- Heading brainstorm: discussion-only answer for `@agent`
- Image placeholder: asset insertion for `@codex`
- Loose request: ambiguous ask for `@agent`, should ask for clarification
- Existing heading thread: active callout with human follow-up
- Existing PR thread: resolved callout with human follow-up

These sections should not be detected:

- Contact example: email address
- Syntax note: inline-code escape
- Terminal paste: quoted source material
- Parked clarification: agent has already yielded and prefilled the human label
- Finished wording thread: sealed resolved thread
- Historical expanded marker: old expanded marker is outside forward V1
- Legacy directive text: legacy hash directive is outside V1
