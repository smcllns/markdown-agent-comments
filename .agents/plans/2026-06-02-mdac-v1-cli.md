# mdac V1 CLI Implementation Plan

Status: in progress

Goal: ship the V1 `mdac` CLI from the approved PRD.

## Scope

- Package: `markdown-agent-comments`
- Binary: `mdac`
- Runtime: portable JS for Node users; Bun for dev/test scripts.
- Commands: `scan`, `run --once`, `watch`.
- Protocol: active `[!NOTE]`, resolved `[!DONE]-`, seal `<!--mdac:eot-->`.
- No legacy `#agent`, no `#silent`, no scheduler, no plugins, no sweep.

## Architecture

- `src/scanner.js`: recursive markdown scan and thread-state detection.
- `src/cli.js`: argument parsing, command dispatch, agent process invocation.
- `test/*.test.js`: Bun tests covering scan, run, watch-adjacent behavior.
- `fixtures/`: current spec fixtures adapted from archived prior art.

## Implementation Notes

- Treat `docs/PRD.md` as source of truth where prior archive disagrees.
- A thread is actionable when:
  - an unwrapped inline `@agent`/`@claude`/`@codex`/custom trigger appears in a markdown line outside blockquote context
  - a `[!NOTE]` callout contains a trigger and the latest real quoted line is human-authored or unsealed non-agent text
  - a `[!DONE]-` callout contains a trigger and the latest real quoted line does not end with `<!--mdac:eot-->`
- If an agent line is latest, the thread is parked even if it ends with a question.
- Label-only human placeholders do not make a thread actionable.
- `run --once` should not invoke an agent when scan is clean.
- Agent command should be overridable for tests and future wrappers.
